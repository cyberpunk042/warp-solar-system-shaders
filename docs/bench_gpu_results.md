# ChromoFold GPU benchmark — reproducible run record

Generated by `python -m warp_compress.bench_gpu`. This file is a **snapshot** of one machine's run so the
numbers are interpretable and reproducible; re-run on your hardware to regenerate. Covers agenda items
**A** (hardware identity + repetition statistics), **B** (four timing layers), **C** (raw GPU gather
baseline), **D** (vocabulary sweep). Items **E** (real-corpus distribution sweep), **F** (RRR-vs-packed-vs-
gzip/zstd frontier), and the shared-prefix prompt-cache LLM experiment are the next round.

## Headline findings (honest)

1. **The user-facing throughput is transfer-bound, not compute-bound.** At V=256, the access *kernel* runs at
   **~1240 M/s** (0.84 ms for 1M random accesses), but the full call including `H2D(positions)` + `D2H(results)`
   over PCIe 3.0 ×16 drops to **~460 M/s**. The CPU round-trip is ~2/3 of the wall time. In a real serving loop
   the positions come from on-GPU attention and the results feed the next on-GPU op, so the **kernel number
   (~1240 M/s) is the relevant one** — and avoiding that round-trip is precisely ChromoFold's thesis.
2. **Raw GPU gather is 5–14× faster** — it's a pure memory op with no rank/search. So at small vocab the
   wavelet must justify itself through rank/search/RRR, not raw access speed.
3. **At LLM-scale vocab the packed wavelet already wins on size:** V=128K → 2.4 B/tok, **1.67× smaller than raw
   uint32**, while remaining searchable (V=32K ≈ uint16 parity). RRR (next round) should push this further.

## What this does and does not establish

- **Establishes:** the raw access kernel is fast and compute-light (~0.8 ns/access device time); the
  user-facing number is dominated by PCIe transfer, which the GPU-resident design exists to avoid; raw gather
  is the honest baseline the wavelet must beat on *function* (rank/search) or *size at large V*, not raw speed.
- **Does not yet establish:** full-pipeline compression vs gzip/zstd/nvCOMP on real corpora (F); an end-to-end
  LLM win (batch-size / TTFT on the shared-prefix prompt cache); or superiority over an optimized production
  gather. Those are the next round, and they're where the thesis is actually won or lost.
